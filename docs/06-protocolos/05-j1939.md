# J1939

## Estado atual

`J1939` é pertinente ao contexto diesel do projeto, mas ainda não aparece como pilha implementada no estado atual do repositório. Não foram encontrados no código analisado parser de `PGN`, banco de `SPN`, gerenciamento de endereço `J1939` ou transporte de mensagens desse protocolo no gateway e no software local.

Assim, a documentação oficial deve tratar J1939 como frente arquitetural ainda não materializada em código executável.

## Funcionamento técnico

A arquitetura existente já fornece blocos reutilizáveis para uma futura integração J1939:

- abstração de sessão e confiabilidade no host;
- gateway como ponto concentrador de transporte e roteamento;
- separação clara entre protocolo do host e protocolo do dispositivo;
- documentação legada com contratos e direcionadores de evolução.

Se uma camada J1939 for incorporada, ela precisará resolver pelo menos quatro responsabilidades técnicas que hoje não estão implementadas:

1. codificação e decodificação de `PGNs`;
2. modelagem dos sinais de aplicação associados;
3. integração com hardware CAN adequado;
4. correlação entre comandos do SimulDIESEL e mensagens J1939.

## Limitações

Não há base factual no repositório para listar `PGNs`, `SPNs`, mensagens periódicas, endereços-fonte ou estratégias de diagnóstico J1939 como se já fossem suportados. Também não existe, no código presente, ponte funcional entre o protocolo host `SGGW` e uma camada J1939 ativa.

## Evolução prevista

A evolução coerente com o restante da arquitetura é que J1939 seja introduzido como uma camada especializada sobre o gateway, e não como acoplamento direto à UI. Quando isso ocorrer, a documentação deverá consolidar:

- matriz de `PGNs` suportados;
- payloads reais por caso de uso;
- conversão entre sinais de módulo e comandos do software local;
- estratégia de teste em bancada com captura de barramento.
