CAN RX atual - CAN fisico -> UCE -> API

Estado atual: polling por UI/API, nao assinc.

1. CAN fisico:
   CanDriver.configureRxMailboxes configura:
   - mailbox 0: STD
   - mailbox 1: EXT
   ambos em CAN_MB_RX_OVER_WR_MODE.

2. Armazenamento:
   Frame recebido fica no mailbox fisico do CAN0.
   Nao ha fila CAN RX em RAM na UCE.

3. Gatilho atual:
   frmUCE_UI._canRxTimer, intervalo 500 ms.
   CanRxTimer_Tick chama PollCanRxAsync somente quando tabDados esta ativa.

4. API:
   frmUCE_UI.PollCanRxAsync
   FrmUceLogic.PollCanRxAsync
   UceDispatcher.PollCanRxAsync
   UceClient.PollCanRxAsync
   SDH Target=UCE.can.rx, Op=poll
   TLV 0x24, payload L=1

5. BPM/SPI:
   Fluxo sincrono normal via GwRouter.route e GwSpiBus.transact.

6. UCE:
   UceTransport.poll
   UceServiceDispatcher.dispatch
   CanService.handleRxPoll
   CanDriver.pollReceived
   CanDriver.readRxMailbox

7. Resposta:
   TLV 0x24:
   controller, count, frame(s) com id big-endian, flags, dlc, data[8].

8. API/UI:
   UceParsers.TryReadCanRxPollResponse
   UceCanRxPollResponse.Frames
   frmUCE_UI adiciona em lstRX.
