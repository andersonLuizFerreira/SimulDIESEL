Contratos TLV CAN atuais

0x20 CAN_CONFIG
Request L=3: controller, bitrateCode, modeCode
Response L=3: controller, acceptedBitrateCode, acceptedModeCode

0x21 CAN_ENABLE
Request L=2: controller, state
Response L=2: controller, effectiveState

0x22 CAN_STATUS
Request L=1: controller
Response L=4: controller, interfaceState, bitrateCode, modeCode

0x23 CAN_RESET
Request L=1: controller
Response L=2: controller, resetStatus

0x24 CAN_RX_POLL
Request L=1: controller
Response L=2+N*14:
  controller
  count
  frame:
    id u32 big-endian
    flags bit0=EXT bit1=RTR
    dlc
    data[8]
Observacao: resposta sincrona, nao evento.

0x25 CAN_DRIVER_LOG_POLL
Request L=1: controller
Response L=2+N*8:
  controller
  count
  entry:
    timestampLow, eventCode, interfaceState, bitrateCode, modeCode, detail0, detail1, detail2

0x26 CAN_TX
Request L=17:
  controller
  flags bit0=EXT bit1=RTR
  dlc
  periodMs uint16 little-endian
  id uint32 little-endian
  data[8]
Response L=3:
  controller
  txStatus
  sequenceOrSlot

0x27 CAN_TX_STOP
Request L=2:
  controller
  slotOrAll: 0x00 ou 0xFF
Response L=2:
  controller
  txStatus

Ausente:
Contrato TLV de evento CAN RX espontaneo.
