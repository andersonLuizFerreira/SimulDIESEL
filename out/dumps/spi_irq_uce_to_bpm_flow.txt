SPI IRQ UCE -> BPM - fluxo verificado

Estado atual:
- UCE SpiLink.write(src,len):
  - zera _txBuf[64]
  - copia src
  - chama pulseAttention()
  - pulseAttention gera pulso no UCE_SPI_IRQ_PIN

- UCE UceTransport:
  - so chama _link.write() dentro de poll()
  - poll() so roda quando _link.available() indica comando recebido da BPM
  - nao ha publish/event publico

- BPM GwSpiBus:
  - transact() usa IRQ da UCE durante comando sincrono
  - primeira sessao envia comando
  - espera IRQ de resposta
  - segunda sessao envia 64 bytes dummy e le resposta
  - nao ha pollEvent() SPI para UCE

- BPM GatewayApp:
  - attachInterrupt existe para GSA IRQ
  - nao existe attachInterrupt para UCE IRQ
  - tick() drena apenas GSA

Conclusao:
- A base eletrica de IRQ na UCE existe.
- A leitura SPI dummy existe embutida no caminho sincrono.
- O caminho assíncrono UCE -> BPM nao esta implementado.
